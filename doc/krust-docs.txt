WHY?                                                         *krust-docs-why?*

To be able to read complete Rust compiler diagnostics in Neovim with proper
colors and formatting, similar to how they appear when running cargo in the
terminal.


INSTALLATION                                         *krust-docs-installation*


LAZY.NVIM ~

>lua
    {
      "alexpasmantier/krust.nvim",
      ft = "rust",
    }
<


PACKER.NVIM ~

>lua
    use {
      "alexpasmantier/krust.nvim",
      ft = "rust",
    }
<


CONFIGURATION                                       *krust-docs-configuration*

**Krust** automatically configures rust-analyzer to send colored diagnostics.
No keybindings are set by default to avoid conflicts.

>lua
    -- With lazy.nvim
    {
      "alexpasmantier/krust.nvim",
      ft = "rust",
      opts = {
        keymap = "<leader>k",  -- Set a keymap for Rust buffers (default: false)
        float_win = {
          border = "rounded",    -- Border style: "none", "single", "double", "rounded", "solid", "shadow"
          auto_focus = false,    -- Auto-focus float (default: false)
        },
      },
    }
<

**Note:** If rust-analyzer starts before krust.nvim loads, you may need
`:LspRestart` for colors to appear. To avoid this, load `krust.nvim` before
your LSP config.


USAGE                                                       *krust-docs-usage*

Use the command:

>vim
    :Krust
<

Or call from Lua:

>lua
    require('krust').render()
<


BEHAVIOR ~

**Krust** tries to behave like LSP hover documentation windows:

- **First invocation**: Opens the floating window (not focused)
- **Second invocation**: Enters the floating window so you may scroll
- **q or <Esc>**: Closes the window
